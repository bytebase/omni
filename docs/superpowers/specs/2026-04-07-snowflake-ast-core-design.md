# Snowflake AST Core (F1) — Design

**DAG node:** F1 — `snowflake/ast` foundational types
**Migration:** `docs/migration/snowflake/dag.md`
**Branch:** `feat/snowflake/ast-core`
**Status:** Approved, ready for plan + implementation.

## Purpose

Establish the foundational AST package for the new hand-written Snowflake parser. F1 defines the `Node` interface, location tracking, and walker infrastructure that every Tier 1+ migration node depends on. It is the root of the Snowflake migration DAG — nothing parser-related can ship until F1 lands.

The package is intentionally minimal: it ships interfaces, base types, and a single root `File` node. All concrete statement, expression, and type nodes are out of scope for F1 and will be contributed by later DAG nodes (T1.x, T2.x, T4.x, T5.x, T7.x).

## Architecture

`snowflake/ast` mirrors `omni/pg/ast` and `omni/mysql/ast` so that the package feels like a sibling to those engines. The pg pattern is the established convention in omni for any engine that will grow to many node types (~600 in Snowflake's case). The Visitor + code-generated `walkChildren` approach is the only thing that scales cleanly to that many node types — hand-rolling traversal for each new node type is a bug farm.

The package omits four PG-internal artifacts that exist in `pg/ast` only because PostgreSQL's C parser emits them: the `List{Items []Node}` wrapper, the `Integer`/`String`/`Float`/`Boolean`/`BitString` value-node wrappers, the `Oid` type, and the `IntList`/`OidList` types. Snowflake comes from a `.g4` grammar, not C — Go primitives in struct fields are more idiomatic and remove a layer of indirection.

The full architectural decision and the alternatives considered are recorded in `docs/migration/snowflake/analysis.md` ("Reference engine") and `docs/migration/snowflake/dag.md`.

## File Layout

```
snowflake/ast/
  node.go              # Node interface, Loc struct, NoLoc, Loc methods
  nodetags.go          # NodeTag enum + initial constants
  loc.go               # NodeLoc switch dispatcher, SpanNodes helper
  walk.go              # Visitor, Walk, Inspect, walkNodes helper
  walk_generated.go    # Generated walkChildren switch (1 case for *File)
  parsenodes.go        # Concrete node types — F1 ships only the File root
  ast_test.go          # Unit tests using *File trees as fixtures
  cmd/genwalker/
    main.go            # Walker code generator, ported from pg/ast/cmd/genwalker
```

Each file has a single, narrow responsibility:

- **`node.go`** — `Node` interface (`Tag() NodeTag`), `Loc` value type with `IsValid`, `Contains`, `Merge` methods, and `NoLoc()` constructor. No production node types.
- **`nodetags.go`** — `NodeTag` int type, `T_Invalid` and `T_File` constants, `String()` for debug output. Tier 1+ contributors append constants here.
- **`loc.go`** — `NodeLoc(n Node) Loc` type-switch dispatcher (initially with one `*File` case plus the default), `SpanNodes(...Node) Loc` variadic span helper. Tier 1+ contributors append cases as they add concrete nodes.
- **`walk.go`** — Hand-written Visitor API. Never code-generated. Defines `Visitor` interface (`Visit(node Node) Visitor`), `Walk`, `Inspect`, the `inspector` func adapter, and `walkNodes(v, []Node)` helper used by the generated `walkChildren`.
- **`walk_generated.go`** — Auto-generated by `cmd/genwalker`. F1 ships it with one case (`*File → walkNodes(v, n.Stmts)`). Never edited by hand. Contributors run `go generate ./snowflake/ast/...` after adding nodes.
- **`parsenodes.go`** — Holds concrete node type definitions. F1 ships only the `File` root container — `type File struct { Stmts []Node; Loc Loc }` — because every parser entry point and every test needs *something* to walk over. All other concrete nodes (statements, expressions, types) are contributed by later DAG nodes.
- **`ast_test.go`** — Unit tests. Uses nested `*File` trees as fixtures so the production `walkChildren` can dispatch on a real production type.
- **`cmd/genwalker/main.go`** — Near-verbatim port of `pg/ast/cmd/genwalker/main.go`. Differences from the pg version: the `*List` branch in `isChildType` is dropped (no `List` wrapper exists in `snowflake/ast`), a `[]Node` branch is added that emits `walkNodes(v, n.Field)`, and the excluded-types list contains only `Loc`. Other slice/pointer/map shapes (`[]*ConcreteStmt`, `*OptionalStmt`, etc.) are added to the generator on demand by later DAG nodes as they introduce fields requiring them.

## Type Definitions

### `node.go`

```go
package ast

// Node is the interface implemented by every snowflake parse-tree node.
type Node interface {
    Tag() NodeTag
}

// Loc represents a source byte range. -1 means "unknown" for either field.
// Loc is a value type embedded as a plain field on every concrete node;
// it is NOT part of the Node interface (mirroring pg/ast).
type Loc struct {
    Start int // inclusive start byte offset (-1 if unknown)
    End   int // exclusive end byte offset   (-1 if unknown)
}

// NoLoc returns a Loc with both fields set to -1 (unknown).
func NoLoc() Loc

// IsValid reports whether both Start and End are non-negative.
func (l Loc) IsValid() bool

// Contains reports whether l fully contains other (inclusive on both ends).
// Returns false if either Loc is invalid.
func (l Loc) Contains(other Loc) bool

// Merge returns the smallest Loc that contains both l and other.
// If either side is invalid, returns the other side.
// If both are invalid, returns NoLoc().
func (l Loc) Merge(other Loc) Loc
```

### `nodetags.go`

```go
package ast

// NodeTag identifies the concrete type of an AST node.
//
// Numeric values are NOT stable — do not persist them. Tags are assigned
// by source order; reorder freely as the package evolves.
type NodeTag int

const (
    T_Invalid NodeTag = iota
    T_File
)

// String returns a human-readable representation of the tag.
func (t NodeTag) String() string
```

### `loc.go`

```go
package ast

// NodeLoc returns the source location of n, or NoLoc() if n is nil or its
// concrete type carries no Loc field. Tier 1+ contributors must add a case
// here for every new concrete node type.
func NodeLoc(n Node) Loc

// SpanNodes returns the smallest Loc that covers every node in nodes.
// Nil entries and nodes with invalid Loc are skipped. Returns NoLoc() if
// no node has a valid Loc.
func SpanNodes(nodes ...Node) Loc
```

### `walk.go`

```go
package ast

//go:generate go run ./cmd/genwalker

// Visitor is implemented by clients of Walk.
//
// Visit is called for each node during a depth-first walk. If Visit returns
// a non-nil w, Walk recurses into the node's children with w, then calls
// w.Visit(nil) to signal end-of-children (post-order). If Visit returns nil,
// children are not visited.
type Visitor interface {
    Visit(node Node) Visitor
}

// Walk traverses an AST in depth-first order.
func Walk(v Visitor, node Node)

// Inspect calls f for every node in the AST in depth-first order.
// If f returns true, Inspect recurses into the node's children.
func Inspect(node Node, f func(Node) bool)

// walkNodes visits each non-nil entry in nodes. Used by walk_generated.go
// to traverse slice-typed child fields like []Node.
func walkNodes(v Visitor, nodes []Node)
```

### `parsenodes.go`

```go
package ast

// File is the root node of a parsed Snowflake source file. It holds the
// top-level statement list and the byte range of the entire file. F4
// (parser-entry) returns *File from Parse.
type File struct {
    Stmts []Node
    Loc   Loc
}

func (f *File) Tag() NodeTag { return T_File }
```

### `walk_generated.go` (initial F1 contents)

```go
// Code generated by genwalker; DO NOT EDIT.

package ast

func walkChildren(v Visitor, node Node) {
    switch n := node.(type) {
    case *File:
        walkNodes(v, n.Stmts)
    }
}
```

## Mutation Strategy

F1 does not introduce a special mutation API. AST rewrites (e.g., LIMIT injection in T3.3) follow the convention used by `pg/ast` and `mysql/deparse`:

1. Walk the tree with `Inspect` to capture a pointer to the target node via closure capture.
2. Mutate the target node's exported fields directly through that pointer.
3. Re-deparse via `snowflake/deparse` (T3.2).

If a future feature requires a parent-aware `Apply`-style cursor, it can ship as a separate function in `walk.go`. Out of scope for F1.

## Testing

Test file: `snowflake/ast/ast_test.go`. Run with `go test ./snowflake/ast/...` (scoped — never global).

Most tests use nested `*File` trees as fixtures so the production `walkChildren` dispatch path is covered. One test (`TestNodeLocUnknown`) defines a private stub type inside the test file to verify the `NodeLoc` default branch — this is the only place in F1 where a non-production node type appears.

### Required test cases

1. **`TestNoLoc`** — `NoLoc()` returns `{-1, -1}`. `IsValid()` is false on it.
2. **`TestLocIsValid`** — Valid loc returns true; loc with `Start = -1` returns false; loc with `End = -1` returns false.
3. **`TestLocContains`** — Proper containment, equal endpoints (inclusive), non-containment (disjoint, partial overlap), invalid loc returns false on both sides.
4. **`TestLocMerge`** — Both valid (correct min/max), one invalid (returns the other), both invalid (returns `NoLoc()`).
5. **`TestNodeLocNil`** — `NodeLoc(nil)` returns `NoLoc()`.
6. **`TestNodeLocFile`** — `NodeLoc(&File{Loc: ...})` returns the File's Loc.
7. **`TestNodeLocUnknown`** — A node whose concrete type has no case in `NodeLoc` returns `NoLoc()`. (Implemented via a private test-local stub type that satisfies `Node` but is intentionally not switched on.)
8. **`TestSpanNodes`** — Variadic with valid+invalid+nil entries returns the merged span over the valid ones; empty input returns `NoLoc()`.
9. **`TestWalkNil`** — `Walk(v, nil)` does not panic and does not call Visit.
10. **`TestWalkVisitOrder`** — Build a 3-deep nested `*File` tree. A recording visitor captures `(node, post)` events. Assert the sequence is pre-order then post-order(nil) at each level.
11. **`TestWalkAbort`** — A visitor that returns nil at the inner File must not visit the leaf File. Records visit events; assert leaf is absent.
12. **`TestInspectShortCircuit`** — `Inspect` callback returning false at the inner File must not visit the leaf. Assert leaf is absent.
13. **`TestNodeTagString`** — `T_Invalid.String()` and `T_File.String()` return their human-readable names; an unknown tag returns "Unknown".
Plus a compile-time assertion at file scope: `var _ Node = (*File)(nil)`.

### Walker generator integrity

Not a Go test, but documented as a contributor expectation: after editing any node file, run `go generate ./snowflake/ast/...`. A future CI check should verify the generated file is up to date by re-running the generator and asserting no diff. F1 does not implement that CI check — it is left to a follow-up task once more nodes exist.

## Out of Scope

F1 does NOT include any of the following — they belong to later DAG nodes:

| Feature | DAG node |
|---------|----------|
| Lexer | F2 |
| Statement splitter | F3 |
| `Parse(sql) (*File, error)` entry point | F4 |
| Concrete statement types (`SelectStmt`, `CreateTableStmt`, …) | T1.4, T2.x, T4.x, T5.x |
| Expression types (`ColumnRef`, `FuncCall`, `BinaryExpr`, …) | T1.3 |
| Data type / type-name nodes | T1.2 |
| Identifier and qualified-name helpers | T1.1 |
| Snowflake Scripting body nodes | T7.1 |
| Deparser | T3.2 |
| LIMIT injection | T3.3 |
| Comment tracking in the AST | Not planned (legacy parser also omits this; pg/ast also omits this) |

## Acceptance Criteria

F1 is complete when:

1. `go build ./snowflake/ast/...` succeeds from the worktree root.
2. `go vet ./snowflake/ast/...` reports no issues.
3. `go test ./snowflake/ast/...` passes all 13 runtime test cases above; the compile-time `var _ Node = (*File)(nil)` assertion holds.
4. `go generate ./snowflake/ast/...` runs cleanly and produces a `walk_generated.go` byte-identical to the committed file.
5. The package compiles in isolation — no Tier 1+ DAG node is required to make F1 build.
6. The DAG document `docs/migration/snowflake/dag.md` has F1's status updated to `done` once merged to `main`.
